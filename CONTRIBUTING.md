# Contributing to Spec Kit Template

Thanks for your interest in contributing!

## How to contribute

- **Issues first (recommended):** For significant changes, open an issue to discuss the approach before coding.
- **Pull requests:** Small fixes and improvements can go directly via a PR.

## Project governance

This repository is maintained by **LKS Next**. Maintainer approval is required to merge changes.

## Development guidelines

- Keep changes focused and well-scoped.
- Follow the existing structure and conventions in `docs/`, `.github/agents/`, `.github/prompts/`, and `.github/skills/`.
- Prefer clear documentation over cleverness: templates should be easy to reuse.
- If you change behavior of prompts/agents/skills, include a short explanation and an example of expected usage.

## Developer Certificate of Origin (DCO)

This project uses the **Developer Certificate of Origin (DCO)** to confirm that contributors have the right to submit their work.

You certify the DCO by adding a `Signed-off-by` line to your commits:

    Signed-off-by: Your Name <your.email@example.com>

### How to sign off commits

Add `-s` to your commit:

    git commit -s -m "your message"

If you have multiple commits in your PR, all commits should be signed off.
