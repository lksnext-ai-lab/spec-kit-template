# TODOs (backlog de especificación)

> Objetivo: lista única de trabajo pendiente de la especificación (no es un backlog de desarrollo).
> Los TODO dentro de documentos están permitidos, pero los relevantes deberían estar reflejados aquí.

## Convenciones

- **ID**: `TODO-###`
- **Estado**: `OPEN` / `IN_PROGRESS` / `DONE` / `DEFERRED`
- **Prioridad**: Alta / Media / Baja
- **Tipo**: Redacción / Validación / Investigación / Decisión / Revisión
- Enlace a archivo/sección: usar ruta y encabezado (ej: `docs/40-arquitectura.md#componentes--servicios`)

## Reglas de uso

- Si un TODO bloquea el avance de una iteración, debe aparecer también en `docs/spec/01-plan.md` como tarea o decisión.
- Si un TODO se cierra, registra el resultado en el documento correspondiente y marca `DONE`.
