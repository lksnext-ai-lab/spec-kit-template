# Preguntas abiertas (OPENQ)

> Objetivo: registrar dudas y decisiones pendientes que impactan la especificación.
> Regla: si una pregunta puede cambiar requisitos/arquitectura, debe estar aquí.

## Convenciones

- **ID**: `OPENQ-###`
- **Estado**: `OPEN` / `ANSWERED` / `WONTFIX` (no aplica) / `DEFERRED`
- **Impacto**: Bajo / Medio / Alto (en alcance, coste, riesgo o plazos)
- Si una respuesta se convierte en decisión técnica relevante: crear ADR y enlazarlo.

## Notas

- `OPEN` debe revisarse en cada iteración (ver `docs/spec/01-plan.md`).
- Cuando una pregunta se resuelva:
  - Actualiza los documentos afectados,
  - Cambia a `ANSWERED`,
  - Y añade el enlace a ADR si aplica.
